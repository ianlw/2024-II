# La industria del software
Resumida en  4 eras:
- Primera Era: 1945-1965 El software estaba pensado únicamente en que hicieran funcionar las computadoras nacientes en aquella época, entonces era prducido por los mismos fabricantes.
- Segunda Era: 1965-1978 Se Comercializa el software con la aparición de desarrolladores independientes, separación del desarrollo de hardware y software.
- Tercera Era: 1978-1993 Las conputadoras se distribuyec a los hogares por li que se popuraliza el software enpaquetado.
- Cuarta Era: 1994-presente: El uso de las computadoras se maximiza y también se interconectan mediante el Internet. Loq ue lleva a una rápiza penetración de mercado con un producto de software.

Surge también el software libre y decodigo abierto, que forman parte crucial del creciminto del software.

# Los proyectos de software
Se desarrollan por equipos de trabajo con mismos objetivos.

Clasificación:
- Exitoso: El proyecto se completa cumplinendo los objetivos.
- Desafiante: El proyecto se conpleta pero no alcanzando todos los objetivos.
- Fracaso: El proyecto se cancela.

Gran parte de los proyectos de software fracasan y con ellos la inversión echa, ya sea pequeña, mediana o grande.

Muchos proyectos que se legan a cumplir terminan siendo no operativos y en algunos casos no se llegan a utilizar

El frazaso de un proyecto se debe a la recurrencia de diferentes fracazos en todos los ámbitos del desarrollo.
Estos fracazos se dan el la implementación, el valor estimado del proyecto y la satisfacción del cliente.
El cliente se niega a pagar una fase preliminar del proyecto donde se contemplarían los riesgos del peoyecto.
Esta fase es la que más influirá en el fracaso del proyecto si se hace mal. De manera consecuente será la que más errores y gastos de reparación genere.

como el producto de un proyecto de software es digital y se vuelve operativo cuando el proyecto termina, entonces no se le da la seriedad necesaria de parte de los clientes.

Los proyectos fracasan porque no se analiza correntamente la factibilidad técnica, los beneficios, costos y tiempos estimados.

El diagnostico de un proyecto muchas veces se pasa por alto debido a querer abaaratar, por parte del cliente, costos, pero esto puede llevar a fracasar el proyecto.

Las metodologías y estándares noo se aplican aorrectamente además qeu los profesionales no son especialistas, po lo general saben un poco de cada cosa.
